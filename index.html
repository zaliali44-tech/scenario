Formula Glossary

Reference
Quick Planning

Uses total workload, effective productive hours, shrinkage, and available HC to estimate staffing and directional SLA.

Required HC ≈ Workload Hours ÷ Productive Hours per HC
Interval Based

Breaks demand into intervals and calculates required HC by interval before aggregating the results.

Total Required HC = aggregate of interval workload and interval capacity
Erlang Style

Uses queueing logic with interval contacts, AHT, answer time target, and target SLA to estimate required agents.

Required HC = minimum agents where Erlang-C SLA ≥ target SLA
HC Gap

Difference between required HC and available productive HC.

HC Gap = Required HC - Available HC
Cost Impact

Estimated additional cost if extra HC is needed to close the positive staffing gap.

Cost Impact = Positive HC Gap × Cost per HC

How to Use / SOP

User Guide
1
Select the Right Engine

Use Quick Planning for leadership scenarios, Interval Based for interval demand analysis, and Erlang Style for queue staffing calculations.

2
Enter or Upload Data

For Interval Based mode, you can paste interval rows, upload CSV, edit the table directly, and sync values back to the text area.

3
Run Simulation

Review required HC, HC gap, service movement, risk, and cost impact.

4
Use the Insights Panel

Check key drivers and recommended actions to understand what needs to change operationally.

Input Guidance

Keep units consistent. AHT should always be in seconds. In Interval Based mode, each row should be Interval,Volume,AHT. Example: 09:00,120,360

Final version with true interval upload and table editing support included. Interval tables can now be uploaded, edited, copied, sorted, and simulated directly.

Recommended Actions

Reduce AHT where possible to improve service resilience.
Improve shrinkage performance where possible.
Use overtime as a temporary control rather than a permanent fix.
